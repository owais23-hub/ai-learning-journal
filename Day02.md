## 🧠 What I Learned

The biggest lesson today wasn't about Python syntax—it was about software engineering.

I realized that writing code that works is different from writing code that is reusable and maintainable.

One important concept I learned is the separation of concerns. Instead of asking for user input and printing results inside every function, professional functions usually receive data as input, process it, and return the result. This makes the code easier to test, reuse, and integrate into larger projects.

I also learned that revisiting fundamentals is not about relearning what I already know. It's about replacing beginner habits with professional engineering practices.
